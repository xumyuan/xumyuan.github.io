类 ​

空白类会占一个字节，这是为了让每个实例的地址都不同，以便区分不同的实例。因为实例化的本质是分配空间。若类中有虚函数，则大小为虚表指针大小。 ==当空白类作为基类时，空白类的大小就优化为0了==

继承 ​

多继承的问题 ​

多继承的二义性问题：

  • 继承同名成员变量

  • 菱形继承 解决办法：

  • 同名成员变量 使用作用域限定符，可以消除成员二义性

    cpp
    class A{
public:
    int a;
};
class B{
public:
    int a;
};
class C:public A,public B{
public:
    void print(){
        A::a=1;
        B::a=2;
    }
};

  • 菱形继承 菱形继承会导致二义性，解决办法是使用虚继承，让最底层的基类成为虚基类，这样就只会有一个基类的实例。

    cpp
    class A{
public:
    int a;
};
class B:virtual public A{
public:
    int b;
};
class C:virtual public A{
public:
    int c;
};
class D:public B,public C{
public:
    int d;
};

虚继承 ​

虚继承中，共同基类的成员只会在最终派生类中保留一份副本，从而避免了多个同名基类子对象的存在。 虚继承的特点：

  • 共享基类：虚继承使得最终派生类中的共同基类子对象在内存中只存在一份，从而避免了冗余。
  • 虚基类表（VTable）：在虚继承中，编译器会为每个虚基类创建一个虚基类表，用来存储虚基类与派生类之间的偏移量等信息。 类调用构造函数的顺序：虚基类->直接基类->类==虚基类只初始化一次==

左值右值 ​

左值：可以取地址的表达式，可以放在赋值符号左边的表达式。 右值：表示即将销毁的临时对象，具有短暂的状态，比如字面值常量'h'，返回非引用类型的表达式int func()等，都会生成右值； 右值引用就是必须绑定到右值的引用，可以通过&&（两个取地址符）来获得右值引用；右值引用只能绑定到即将销毁的对象，因此可以自由地移动其资源。 右值引用的作用： 右值引用是为了支持移动操作而引出的一个概念，它只能绑定到一个将要销毁的对象，使用右值引用的移动操作可以避免无谓的拷贝，提高性能。使用std::move()函数可以将一个左值转换为右值引用。（可以通过两个很长的字符串的直接赋值和移动赋值来测试一下性能的差距）。 std::move底层只是做了一个static_cast<T&&>的操作，将左值转换为右值引用。

==一个声明的右值引用其实是一个左值==

移动语义 ​

传统 C++ 通过拷贝构造函数和赋值操作符为类对象设计了拷贝/复制的概念，但为了实现对资源的移动操作， 调用者必须使用先复制、再析构的方式，浪费了很多性能。 直接通过右值引用，接管将亡值的资源管理，延长生命周期。

完美转发 ​

由于声明的右值引用实际上是一个左值，因此在进行参数传递时会出现问题，这时候就需要使用完美转发。

cpp
void reference(int &v){
    std::cout << "左值引用" << std::endl;
}
void reference(int &&v){
    std::cout << "右值引用" << std::endl;
}
void pass(int &&v){
    std::cout<<"普通传参";
    reference(v);
    std::cout<<"std::move传参";
    reference(std::move(v));
    std::cout<<"std::forward传参";
    reference(std::forward<int>(v));
    std::cout<<"static_cast传参";
    reference(static_cast<int&&>(v));
    
}
int main() {
    std::cout << "传递右值:" << std::endl;
    pass(1);

    std::cout << "传递左值:" << std::endl;
    int v = 1;
    pass(v);

    return 0;
}

上述代码的运行结果为：

powershell
传递右值:
              普通传参: 左值引用
       std::move 传参: 右值引用
    std::forward 传参: 右值引用
static_cast<T&&> 传参: 右值引用
传递左值:
              普通传参: 左值引用
       std::move 传参: 右值引用
    std::forward 传参: 左值引用
static_cast<T&&> 传参: 左值引用

std::forward即没有造成任何多余的拷贝，同时完美转发(传递)了函数的实参给了内部调用的其他函数。

内存模型 ​

C++内存分为五个区

  • 栈：为函数的局部变量分配内存，能分配较小的内存；
  • 堆，使用malloc、free动态分配和释放空间，能分配较大的内存；
  • 全局/静态存储区，用于存储全局变量和静态变量；
  • 常量存储区，专门用来存放常量；
  • 代码区；

栈比堆速度快的原因：

  • ==申请速度快==（最主要原因）：栈是程序运行前就已经分配好的空间，所以运行时分配几乎不需要时间。(栈上的申请动作，几乎只需要一条cpu指令，但是堆会经过一些列软件操作)而堆是运行时动态申请的，相当于将分配内存的耗时由编译阶段转嫁到了机器运行阶段，将分配过程从编译器搬到了运行的代码中。于是动态分配的速度不仅与分配算法有关，还与机器运行速度有关。（栈是编译时分配空间，而堆是动态分配（运行时分配空间），所以栈的申请速度快）
  • 存储寻址速度快：栈的物理地址空间是连续的，而堆未必，查找堆的链表也会耗费较多时间，所以存储寻址速度慢。
  • CPU硬件操作速度快：cpu有专门的寄存器（esp，ebp）来操作栈，堆是使用间接寻址的，所以栈快。 （如果能预先分配好空间，堆上操作在这片空间里，两者速度不会差距太大）

精准控制内存的操作：

  • 位域
cpp
struct Example{
    unsigned int data : 31;
    unsigned int isVisited : 1;
}

可以位域来进行更精准的内存控制，上述代码指第一个参数占用31位，第二个占用1位。 PS：C99规定int、unsigned int、bool可以做位域的类型

内联函数 ​

内联函数的优缺点

  • 优点：在编译的时候，将调用内联的地方直接将内联函数的代码块替换，节省了函数调用带来的开销。
  • 缺点：可能会造成代码的膨胀，编译时间比较缓慢，造成比较大内存开销，exe比较大，占用CPU资源

内联函数和宏的区别：

  • 内联函数是由编译器处理的，宏是由预处理器处理的。
  • 内联函数会进行类型检查，宏不会。

内联函数和普通函数的区别 inline函数与普通函数的区别在于，当编译器处理调用内联函数的语句时，不会将该语句编译成函数调用的指令，而是直接将整个函数体的代码插人调用语句处，就像整个函数体在调用处被重写了一遍一样。这一点有点类似于宏定义。中心思想是以空间换时间

类型双关 ​

简单来说就是可以直接按照想要的方式访问内存中的数据。

cpp
int a=10;
double b=a; //这里会发生隐式转换
double c=*(double*)&a; //这里会发生类型双关,直接访问内存中的数据，但是并不安全，这里两类型大小不一致，可能导致错误

union ​

可以在一个union中定义多种不同的数据类型，这些数据共享一块内存，但是只能同时使用其中的一种。union的大小是其最大成员的大小。

cpp
struct Vector4{
    union{
        struct{
            float x,y,z,w;
        };
        float elements[4];
        //这里的两块数据共享一段内存
    };
};

类型转换 ​

c++中的类型转换有四种：

  • static_cast：用于非多态类型的转换，如基本数据类型之间的转换，父类指针转换为子类指针，void指针转换为其他类型指针等。编译时转换
  • dynamic_cast：用于多态类型(包含虚函数的类)的转换，只能用于含有虚函数的类，用于将父类指针转换为子类指针。运行时类型转换，会进行运行时的类型检查。
  • const_cast：用于去除const属性。
  • reinterpret_cast：用于将一个指针转换为一个不同类型的指针，如将int指针转换为char指针。

static_cast和dynamic_cast的区别：

  • 从派生类转到基类是合法的，此时两种方式效果都是一样的。
  • 在进行基类向派生类的转换时，dynamic_cast具有类型检查功能，比static_cast更安全
  • static_cast是在编译时转换，dynamic_cast是在运行时转换。 这里base_ptr只是单纯的父类指针，并不包含子类成员a，做dynamic_cast转换时，会报错。

这里是否能成功转换就是指：判断指针是否真的是子类指针，如果被转换的指针只是基类指针，可能并不包含子类成员，这是dynamic_cast转换会报错。

构造函数 ​

禁止类被继承 ​

  • 构造函数与析构函数设置为私有。 把父类的构造函数和析构函数定义为私有成员，此时子类便不可调用父类的构造函数和析构函数，因此便达成了防止一个类被继承的目的，但是此时又出现一个问题，如何去构造一个类对象呢。 可以写一个共有的静态的函数来返回一个对象，这种方式和只能在堆上建立对象是一样的。

  • 使用虚拟继承 当我们使用虚拟继承构造一个类的时候，这样就会使得一个类可以在堆上建立对象也可以在栈上建立对象，这样也避免了方法一中出现的无法在栈上建立对象了。 下面的写法可以实现MyClass的禁止继承

cpp
#include <iostream>
using namespace std;

template <class T>
class Base
{
	friend T;//模板参数类型作为友元函数
private:
	Base()
	{}
	~Base()
	{}
};
//FinalClass作为Base的友元，可以访问Base的私有成员
class MyClass :virtual public Base<MyClass>
{
public:
	MyClass()
	{}
	~MyClass()
	{}
};
//TestClass继承于Base类
//由于是虚拟继承，派生类会访问基类的构造函数和析构函数
//但是由于基类的构造函数是私有所以无法访问
class TestClass : public MyClass
{
public:
    TestClass()
    {}//这里会报错
    ~TestClass()
    {}//这里会报错
};
int main()
{
	MyClass* p = new MyClass;
	MyClass fc;
	//TestClass tc;
	return 0;
}

因为在构造对象的时候，由于MyClass是虚拟继承，派生类会直接访问基类也就是Base类的构造函数和析构函数，而Base类里面的构造函数和析构函数是私有的，所以无法继承。

  • 上述方法，本质是禁止派生类被实例化，要想实现真正的禁止继承。C++11标准下,使用final关键词可以禁止类被继承。

成员初始化列表 ​

cpp
class Entity{
private:
    int a;
    std::string b;
public:
    Entity()
        :a(0),b("Unknown")
    {}
    Entity(int a,std::string b)
        :a(a),b(b)
    {}
}

使用成员初始化列表可以避免对象的重复构造，如果在构造函数对成员惊醒初始化可能会造成性能损失，考虑下面的定义方法：

cpp
class Entity{
private:
    int a;
    std::string b;
public:
    Entity()
    {
        a=0;
        b="Unknown"; //这里实际等价于b=std::string("Unknown");
    }
    Entity(int a,std::string b)
        :a(a),b(b)
    {}
}

类在实例化时，就会对成员变量进行定义，不使用成员初始化列表的方法等价于：

cpp
std::string b;
b=std::string("Unknown");

重复调用了string的构造，使用成员初始化列表的的定义等价于：

cpp
std::string b="Unknown";

只会调用一次，因此应尽可能的使用成员初始化列表。 同时成员初始化列表还能保持代码的简洁性，可以在构造函数中书写更简洁的初始化代码。

实例化对象 ​

使用new关键字实例化的对象在堆上，不使用的在栈上。

cpp
Entity e;   //在栈上
Entity *e=new Entity();  //在堆上，关键是new关键字

初始化在栈上是最简单的初始化方式，生命周期伴随函数结束。如果没有其他需求，尽量这样使用。 不适用栈分配的情况：

  • 要想让对象离开了当前作用域仍然存在，想显式的控制对象
  • 对象占用空间很大，而且可能会创建多个，栈空间比较小，一般一两MB。 初始化在堆上会一直存在，直到释放内存。

new ​

new是一个操作符。new关键字的作用就是找到一个满足要求的内存空间，然后返回其内存地址，这些操作在堆上完成。使用new之后一定要有对应的delete。 通常new会调用malloc。

cpp
Entity* e=new Entity();
Entity* e=(Entity*)malloc(sizeof(Entity)); //尽量不要这么写

这两行代码的区别就是new会调用构造函数。

auto ​

拷贝构造函数 ​

下面的操作实际上在做浅拷贝，想实现深拷贝，可以通过拷贝构造函数来实现，在进行函数调用值传递的过程中，也会对拷贝构造函数进行调用。因此，使用传递常引用来对只读的对象进行函数传参。 但当数据成员中有指针时，如果采用简单的浅拷贝，则两类中的两个指针将指向同一个地址，当对象快结束时，会调用两次析构函数，而导致指针悬挂现象。所以，这时，必须采用深拷贝。 当出现类的等号赋值时，会调用拷贝函数，在未定义显示拷贝构造函数的情况下，系统会调用默认的拷贝函数——即浅拷贝，它能够完成成员的一一复制。当数据成员中没有指针时，浅拷贝是可行的。 拷贝构造函数和赋值函数的区别： 判断方式：如果临时变量是第一次出现，那么调用的只能是拷贝构造函数，反之如果变量已经存在，那么调用的就是赋值函数。

const ​

实际上底层并没有做什么，是对开发人员的限制。

  • 顶层const：指针本身是一个常量。
  • 底层const：指针所指向的对象是一个常量。
cpp
int i=0;
int *const p1=&i;       //p1的值不能改变，这是顶层const
const int ci=42;        //ci的值不能改变，这是顶层const
const int *p2=&ci;      //p2可以改变，p2指向的ci不可改变，这是底层const
const int* const p3=p2; //左侧的const是底层const，右侧是顶层const
const int &r=ci;        //用于声明引用的const都是底层const

用法：

  • 总体规则是const默认与左边结合，左边没有东西则与右边结合。const int* aconst与int结合，变量a是一个指向常量整型的指针。int const * a也一样。int* const aconst与*结合变量a是一个指向整型的常量指针。
  • 类中const成员函数和非const成员函数可以构成重载
  • 在类中的成员方法后，表示该方法不会修改任何成员变量。
  • const类对象只能调用const成员函数，非const对象都能调用，都存在时，会优先调用非const函数。
  • const成员函数可以修改mutalbe关键字修饰的成员变量。

static ​

用法：

  • 类外：在链接阶段是局部的，只服务于这个文件。类似在类中定义的私有变量。
  • 在函数中，静态局部变量的生命周期是全局的。多次调用该函数也只会初始化一次。
  • 类中：成员属于类而不是对象

多态 ​

所谓多态，就是同一个函数名具有多种状态，或者说一个接口具有不同的行为；C++的多态分为编译时多态和运行时多态

  • 编译时多态也称为为静态联编，通过重载和模板来实现
  • 运行时多态称为动态联编，通过继承和虚函数来实现

虚函数 ​

虚函数的实现机制：

  • 虚函数是通过虚函数表来实现的，虚函数表包含了一个类(所有)的虚函数的地址，在有虚函数的类对象中，它内存空间的头部会有一个虚函数表指针(虚表指针)，用来管理虚函数表。当子类对象对父类虚函数进行重写的时候，虚函数表的相应虚函数地址会发生改变，改写成这个虚函数的地址，当我们用一个父类的指针来操作子类对象的时候，它可以指明实际所调用的函数。
  • 如果子类中不存在虚函数，那么子类的虚指针将直接指向父类的虚函数表，如果子类中有虚函数，则会产生自己的虚函数表；如果没有将父类中的虚函数全部重载，那么没有被重载的虚函数指针将会指向父类虚函数的地址。

虚函数调用是在编译时确定还是运行时确定的？如何确定调用哪个函数？

  • 关键在于指针指向的类是谁，这决定了去查哪个类的虚函数表
  • 只有通过指针或者引用的方式调用虚函数是运行时确定，通过值调用的虚函数是编译期就可以确定的

构造函数不能是虚函数，因为调用虚函数需要虚函数表，需要有虚函数表指针，而虚函数表指针是在构造函数中初始化的，因此构造函数不能是虚函数。 析构函数常常是虚函数，因为在析构的时候，如果不是虚函数，可能会造成内存泄漏。

cpp
class Base{
}

class Derived:public Base{
}

Base* b=new Derived();
delete b; //如果析构函数不是虚函数，只会调用Base的析构函数，而Derived的析构函数不会被调用，可能造成内存泄漏

虚函数存在于类中，不同的类对象共享一张虚函数表(为了节省内存空间)。虚函数表在编译期就创建好了。 虚函数表指针在对象创建时创建。

纯虚函数 ​

纯虚函数的用法类似其他语言（如java）中的接口，包含纯虚函数的类不能被实例化。纯虚函数在子类必须被实现。

数组 ​

使用new关键字创建的数组是在堆上的，需要手动释放，不适用的则是在栈上创建的，与函数生命周期相同。 std::array最大的好处是支持迭代器，不用手动维护数组大小。并且有边界检查。 实际他并不存储大小，创建时需要指定大小,size()只是返回了这个大小，并没有存储开销。 大部分情况下，和默认的数据性能一致，优于vector。

设计模式 ​

单例模式 ​

保证一个类仅有一个实例，并提供一个访问它的全局访问点，该实例被所有程序模块共享。 定义一个单例类：

  1. 私有化它的构造函数，以防止外界创建单例类的对象；
  2. 使用静态局部变量定义类的唯一实例；
  3. 使用一个公有的静态方法获取该实例。

懒汉式 ​

单例实例在第一次被使用时才进行初始化，这叫做延迟初始化。线程不安全，需要加锁，并且不能简单的在前面判空加锁，因为可能某个线程正在初始化单例，另一个线程却在判断单例是否为空，这样就会获取到不完全的单例，造成错误。

cpp
class Singleton  
{ 
public:  
    static Singleton* getInstance() {
        //第一个检查，如果只是读操作，就不许用加锁
        if (m_pInstance == nullptr) {
            std::lock_guard<std::mutex> lck(m_mutex);
            //第二个检查，保证只有一个
            if (m_pInstance == nullptr) {
                m_pInstance = new Singleton;
            }
        }
        return m_pInstance;
    }
    static void delInstance(){ // 为了多线程安全，本人觉得释放操作也要做 double-check
    	if(m_pInstance != nullptr) 
    	{
    		std::lock_guard<std::mutex> lck(m_mutex);
    		if(m_pInstance != nullptr) 
    		{
            	delete m_pInstance;
            	m_pInstance = nullptr;
        	}
    	}        
    };
private:
    Singleton(){
        std::cout << "Singleton Hello" << std::endl;
    };
    ~Singleton() { // 私有化 可以避免 直接 delete s1 ，必须 使用 delInstance
        std::cout << "Singleton Bye" << std::endl;
    }
    static Singleton* m_pInstance;
    static std::mutex m_mutex;
};  
Singleton* Singleton::m_pInstance = nullptr;
std::mutex Singleton::m_mutex;

在C++11标准下，静态局部变量的初始化是线程安全的，所以不需要额外加锁保证线程安全性。

cpp
class Singleton {
public:
    //禁止拷贝构造函数和赋值运算符重载，确保只有一个实例
    Singleton(const Singleton& singleton) = delete;  
    Singleton(const Singleton&& singleton) = delete; 

    Singleton& operator=(const Singleton& singleton) = delete;
    Singleton& operator=(const Singleton&& singleton) = delete;

    //获取实例的静态方法
    static Singleton& getInstance() {
        static Singleton instance;
        return instance;
    }

protected:
	Singleton() = default; //构造函数私有化
	virtual ~Singleton() = default;
};

饿汉式 ​

饿汉式单例模式是指单例实例在程序运行时被立即执行初始化

cpp
class Singleton
{
public:
	static Singleton* GetInstance()
	{
		return &m_instance;
	}
private:
	// 构造函数私有
	Singleton(){};
	// C++98 防拷贝
	Singleton(Singleton const&);
	Singleton& operator=(Singleton const&);
	// or
	// C++11
	Singleton(Singleton const&) = delete;
	Singleton& operator=(Singleton const&) = delete;
	static Singleton m_instance;
};
Singleton Singleton::m_instance; // 在程序入口之前就完成单例对象的初始化

静态成员 ​

函数内静态成员变量 or 类的静态成员变量？ 因为如果静态类之间有依赖，可能会导致C++的一些未定义的行为。

  • 类的静态成员变量SingletonA::instance保存在程序全局的静态数据区，instance初始化的时机是在程序的main()函数执行前。
  • 假设有另一个单例类与现有类的定义相似，也是类的静态成员变量。SingletonA::instance和SingletonB::instance的初始化顺序是未定义的，看编译器。
  • 如果SingletonA::instance 的初始化在SingletonB::instance之前，而SingletonA的构造函数中恰好需要引用到SingletonB::instance，就很可能会出现一些未定义的行为。

快速排序 ​

最好时间复杂度：O(nlogn) 最坏时间复杂度：O(n^2) 平均时间复杂度：O(nlogn) 最坏时间复杂度出现的情况：数组已经有序，每次选取的基准值都是最大或最小值。 空间复杂度：O(logn)，递归调用的栈空间

快排vs堆排 ​

堆排序是稳定的nlogn，而快排会有好坏的情况。但是常用快排而不是堆排的主要原因是堆排序的数据比较一般不会发生在相邻的数组元素，cache的命中率很低，使得数据读取开销更大
