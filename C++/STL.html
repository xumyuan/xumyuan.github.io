push_back和emplace_back

push_back() 向容器尾部添加元素时，首先会创建这个元素，然后再将这个元素拷贝或者移动到容器中（如果是拷贝的话，事后会自行销毁先前创建的这个元素）

而 emplace_back() 在实现时，则是直接在容器尾部创建这个元素，省去了拷贝或移动元素的过程。

C++
struct A {
        int num;

        A():num(0){}
        A(int a) :num(a) {
                cout << "here is constructor" << endl;
        }

        A(const A& a):num(a.num) {
                std::cout << "here is copy constructor" << std::endl;
        }

        A(A &&a):num(a.num){
                std::cout << "here is move constructor" << endl;
        }

        A& operator=(const A& a) {
                std::cout << "here is =" << std::endl;
                return *this;
        }
        ~A() {
                cout << "here is destructor" << endl;
        }

};

void test_insert() {
        vector<A> test;
        test.reserve(10);

        cout << "push_back test" << endl;
        test.push_back(2);
        cout << "----------------------------"<<endl;
        cout << "emplace_back test" << endl;
        test.emplace_back(2);

        cout << "----------------test over-------------" << endl;
}

push_back只接受一个参数，并且不能接受构造函数的参数，只能接受指定的对象。这里可以使用test.push_back(2)是隐式构造函数，如果在构造函数前加上explicit关键字就会报错。

运行结果如下：

push_back先创建了元素，然后移动到了容器中。

emplace_back直接在容器空间构造了元素

如果注释掉了移动构造函数，运行结果如下：

push_back的底层实现，会优先调用移动构造函数，如果没有则调用拷贝构造函数。

这里push_back会有一次析构，是因为创建了临时变量，然后销毁了

上述实现是二者接受的参数都是构造参数的情况，在这种情况下，emplace_back显然更优。

如果插入已存在的对象，也可以说是左值。

C++
void test_insert() {
        vector<A> test;
        A a(2);
        test.reserve(5);

        cout << "push_back test" << endl;
        test.push_back(a);
        cout << "----------------------------"<<endl;
        cout << "emplace_back test" << endl;
        test.emplace_back(a);

        cout << "----------------test over-------------" << endl;
}

扩容 ​

当向vector中插入元素时，如果元素有效个数size与空间容量capacity相等时，vector内部会触发扩容机制：

  • 开辟新空间
  • 拷贝元素
  • 释放旧空间

msvc是按照1.5倍进行扩容，gcc是两倍。

1.5倍 vs 2倍 ​

1.5倍扩容更优，在假定释放的空间会合并的情况下，2倍扩容每次扩容时，之前合并的空间都不够重新分配，但是1.5倍扩容只需要几次，就能使用之前释放的空间了。

这个只是假定情况，实际情况是否有效存疑

回收 ​

如果一个容器的容量给到了很大的值，但是由于逻辑需要，resize为较小的值，这时会产生很大的空间浪费。可以通过swap来进行内存回收。

具体用法如下：

C++
vector<type>(v).swap(v);

这个用法分为两个部分vector(v)和.swap(v)，前者创建了匿名对象v，他拥有v的全部数据，但是没有冗余空间，即他的容量和大小是一致的；后者则通过swap将其和v交换了内容。此时的v就是没有冗余空间的数组。以此实现空间回收的功能。

C++
void test_expansion() {
        vector<int> a;
        size_t capacity = a.capacity();
        for (int i = 0; i < 500;i++) {
                a.emplace_back(i);
                if (capacity != a.capacity()) {
                        capacity = a.capacity();
                        std::cout << "capacity:" << capacity << endl;
                }
        }

        a.resize(5);
        cout <<"after resize:" << a.capacity() << endl;
        vector<int>(a).swap(a);
        cout << "after swap:" << a.capacity() << endl;

}

优化 ​

简单的优化方案:

  • 设置vector大小,使用vec.reserve(3);在构造函数中传入大小会默认构造这个数目的空对象，reserve不会构造对象。
  • 使用emplace_back() 函数可以减少一次拷贝或移动构造的过程，提升容器插入数据的效率。
C++
vec.push_back(Vec3(x,y,z));
vec.emplace_back(x,y,z);//速度更快，应更多的使用这个

map和set ​

map和set ​

map和set底层是红黑树，红黑树是一种二叉搜索树。

红黑树和(平衡二叉树)AVL树的区别：AVL树是高度平衡的，频繁的插入和删除回导致频繁的旋转，导致效率下降；红黑树不是高度平衡的，算是一种折中，插入最多两次旋转，删除最多三次旋转。

unordered_map和unordered_set ​

底层是基于哈希表实现的，是无序的。理论上增删查改的时间复杂度是O(1)(最差时间复杂度O(n))，实际上数据的分布是否均匀会极大影响容器的性能。

复杂对象作为map的键 ​

对于复杂的对象，并不能直接将类型作为map的键，需要自定义hash函数和判等方法。

C++
struct A {
        int a;
        float b;
};

要想让A作为map的键，要实现它的哈希函数和判等方法。可以重载std::hash和重载==运算符。

C++
struct A {
        int a;
        char b;
        B* c;
        bool operator==(const A& other) const {
                return a == other.a && b == other.b;
        }
};

namespace std {
        template<>
        struct hash<A> {
                size_t operator()(const A& a) const {
                        return hash<int>()(a.a) ^ hash<char>()(a.b);
                }
        };
}

也可以自定义函数后，作为模板参数传递给map

C++
struct Hashfunc {
        size_t operator() (const A& key) const {
                return hash<int>()(key.a) ^ hash<char>()(key.b);
        }
};
struct Equalfunc {
        bool operator() (const A& a, const A& b) const {
                return a.a == b.a && a.b == b.b;
        }
};

unordered_map<A, B, Hashfunc, Equalfunc> map;

这里的用法是仿函数。调用方法为HashFunc(A(a))。
