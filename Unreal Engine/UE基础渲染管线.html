UE基础渲染管线

管线总览 ​

一个最基础的三人称新项目的渲染管线如下：

（图中没有任何复杂的用例，只有空项目的基础场景，采用UE4.27版本截帧）

Z-prepass ​

prepass就是z-prepass，将不透明的物体渲染到深度缓冲

采用reverse-Z保存深度，近裁面的深度为1（reverse-Z是为了保证远处不发生zfighting，因为float靠近0处的精度更高）

这里有一个Forced by DBuffer的描述，DBuffer好像是ue用来保存延迟贴画的缓冲，暂时还不太懂贴花，这里暂时不深究。总之z-prepass还在别的地方有用到。

ComputeLightGrid ​

这里是为了实现Clustered Deferred Shading，计算出会受到局部光源影响的tile，然后在一次pass中处理所有的光源。

BeginOcclusionTests 遮挡测试 ​

BeginOcclusionTests主要是为了处理一帧中的所有遮挡测试。

UE这边主要的实现思路大体是这样的：

  1. 将所有被标记为遮挡体的物理渲染进一个深度缓冲。（这一步渲染的是会遮挡其他物体的mesh）
  2. 创建一个遮挡查询，这一步会提交希望查询遮挡情况的模型，使用z-test和1中的深度缓冲，遮挡查询返回通过深度测试的像素数量。如果是0表示完全被遮挡。这一步可能的渲染开销很高，所以采用包围盒而不是原模型。这一步渲染的是可能被遮挡体遮挡的物体
  3. 回读查询结果到cpu，然后根据返回的像素数量来决定是否将模型提交给gpu渲染

这里可以看到mesh都是包围盒

这里关闭了深度写入和颜色写入，所以renderdoc里看不到具体的信息。

这里先是判断了通过深度查询的像素数量是否为0，然后计算了一个遮挡像素的比例，计算的是占据整个屏幕的比例（这样的好处是可以轻松计算回实际的像素数量？也不用存储屏幕空间中物体包围盒的像素数），一些占比抄底的像素可能也可以直接丢掉。

关于这里LastPixelsPercentage 的后续用途，这里有一段逻辑来判断是否需要后续加入遮挡查询（因为这里的遮挡判断不是每一帧都会有的，可能是间隔几帧一次）

这里的逻辑大致如下：

  1. 判断是否进行精确遮挡查询
  2. 若为精确遮挡查询，会判断这个通过深度测试比例和阈值进行比较，最大值为1，然后将这个值乘上一个随机数再和这个阈值进行比较，来判断是否要进行遮挡测试（这里MaxOcclusionPixelsFraction = 0.1）
  3. 非精确查询则比较按照上次被判定为可见的时间和当前时间和一个可能的时间阈值乘随机数来比较，他核心的原理就是认为如果上一帧可见，邻近的下一帧应该也是可见的，相隔的越远，需要再次深度测试的概率就越大

BuildHZB ​

这里就是生成一个HiZ的Buffer，用于遮挡的粗剔

这里会按照2的幂次取整，便于生成计算

ShadowDepth ​

这一步是用来生成shadowmap

BasePass ​

这里生成延迟渲染所需要的GBuffer，来避免多余的光照计算。正常来说天空球应该是最后渲染的，但是由于有z-prepass，所以也避免了天空球的重绘。

GBuffer的结构大致如下：

  • SceneColorDeferred：物体间接照明的数据
  • GBufferA：世界坐标下的法向量
  • GBufferB：材质属性（金属度、粗糙度、高光强度）
  • GBufferC：RGB通道中保存了反照率（Albedo），alpha通道中保存了环境光遮蔽（AO）系数
  • GBufferD：预烘焙的阴影值
  • GBufferE：由光照模型所决定的自定义数据（例如表面下散射的颜色，切空间向量等等）

Velocity ​

速度值渲染，这里是用来做运动模糊，或者需要重投影操作（reprojection）的特效上（例如Temporal AA）

速度的计算很简单，是基于屏幕空间的算法，用这一帧的屏幕空间位置和上一帧的做差，加上深度，就可以得到三维的速度向量

Java
float3 Calculate3DVelocity(float4 PackedVelocityA, float4 PackedVelocityC)
{
    // 当前帧屏幕位置（去除 TAA Jitter）
    float2 ScreenPos = PackedVelocityA.xy / PackedVelocityA.w - ResolvedView.TemporalAAJitter.xy;
    
    // 上一帧屏幕位置（去除 TAA Jitter）
    float2 PrevScreenPos = PackedVelocityC.xy / PackedVelocityC.w - ResolvedView.TemporalAAJitter.zw;
    
    // 当前帧和上一帧的深度
    float DeviceZ = PackedVelocityA.z / PackedVelocityA.w;
    float PrevDeviceZ = PackedVelocityC.z / PackedVelocityC.w;
    
    // 3D 速度 = (屏幕空间 XY 位移, 深度 Z 位移)
    float3 Velocity = float3(ScreenPos - PrevScreenPos, DeviceZ - PrevDeviceZ);
    
    return Velocity;
}
